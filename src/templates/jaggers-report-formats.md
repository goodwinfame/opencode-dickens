# Jaggers 输出格式模板

本文件包含 Jaggers 审校和设计评审的所有输出格式模板。Jaggers 在产出报告时读取本文件中的对应模板。

---

## 第一轮：AI味检测 — 输出格式

```
## 第一轮：AI味检测

### 结果：通过 / 需修改

### 发现的问题
1. [第X段] 使用了"不禁"——建议改为 [具体修改建议]
2. [第X-Y段] 三个段落结构完全一致（描写→反应→总结）
3. [角色A和角色B的对话] 语气完全相同，无法区分
...

### 问题统计
- 黑名单词汇：X 处
- 结构重复：X 处
- 对话同质化：X 处
- 情感平滑化：X 处

### 总体AI味评级：低 / 中 / 高
```

---

## 第二轮：逻辑一致性 — 输出格式

```
## 第二轮：逻辑一致性

### 结果：通过 / 需修改

### 一致性问题
1. [问题描述] — 严重程度：高/中/低
   - 事实：[章节中写的]
   - 应该是：[根据档案/前文/设定应该是]
   - 修改建议：[具体建议]
...

### 与弧段计划的偏差
- 计划要求：[X]
- 实际执行：[Y]
- 是否可接受：[是/否，原因]
```

## 第三轮：文学性评审 — 输出格式

```
## 第三轮：文学性评审

### 评分
| 维度 | 分数 | 备注 |
|------|------|------|
| 文笔质量 | X/10 | ... |
| 情感冲击力 | X/10 | ... |
| 节奏控制 | X/10 | ... |
| 角色鲜活度 | X/10 | ... |
| 章末效果 | X/10 | ... |
| **综合** | **X/10** | |

### 亮点
- [值得保留的精彩段落或技巧]

### 润色建议
1. [具体建议，带引用原文]
2. ...

### 判定
- 综合 ≥ 7：通过
- 综合 5-6：建议润色（列出具体段落）
- 综合 ≤ 4：需要重点修改
```

## 三轮总结 — 输出格式

```
## 审校总结：第 [N] 章 "[标题]"

### 最终判定：✅ 通过 / ⚠️ 需修改 / ❌ 需重写

### 需修改的问题（按优先级排序）
1. [最重要的问题]
2. ...

### 发给 @weller 的修改指令
[如果需要修改，写出清晰的修改指令，精确到段落和修改方向]
```
