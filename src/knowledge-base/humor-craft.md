# 幽默写作技法

本文档解决 AI 写作中"笑话不好笑"的核心问题。幽默不是随机的，它有可操作的结构。

---

## 一、为什么 AI 写的笑话不好笑？

### 反面模式清单

**冷笑话综合症**：笑话结构正确但缺少情境。谐音梗没有画面感、吐槽没有情绪温度、反转太小不痛不痒。

**分析型吐槽**：角色的吐槽像在做逻辑分析而不是真的吐槽。"这不合理，因为……"——这是分析，不是幽默。真正的吐槽是："行吧，我上辈子大概是个靶子。"

**笑点脱节**：为了搞笑而搞笑，在紧张或感人的场景中突然插入不合时宜的笑话，打断叙事节奏。

**全员同款幽默**：所有角色的笑点来源和表达方式一模一样。冷面角色和话痨角色用同样的语气吐槽。

**过度解释**：笑话讲完后又解释一遍为什么好笑。笑话一旦被解释就死了。

## 二、核心搞笑机制

### 机制 1：反差

**角色反差**：自带矛盾属性的人设。
- 一本正经的武道宗师，养了一只叫"小甜甜"的猛虎
- 以冷酷著称的杀手，有严重的路痴，每次暗杀都要先迷路半小时
- 天才少女，生活自理能力为零，袜子永远穿反

**场景反差**：严肃场景中的日常化处理。
- 用外交辞令讨论谁洗碗
- 两个绝世高手过招，旁白的关注点在于他们脚下踩烂了一个老农的西瓜田

**身份反差**：角色处于与身份完全不搭的情境中。
- 魔王在人间打工，被店长批评服务态度差
- 修仙界第一美人，吃相极其难看

### 机制 2：反转（预期违背）

核心结构：**铺垫预期 → 打破预期**。预期建立得越稳固，打破时越好笑。

**误导式反转**：先让读者以为A，再揭示其实是B。
```
他深吸一口气，做出了一生中最艰难的决定——
点了一份加辣的。
```

**真香式反转**：角色信誓旦旦说绝不做某事，然后下一秒就做了。关键是角色**自己也知道**自己打脸了，但死不承认。

**升级式反转**：情况已经很糟了，然后变得更糟，再更糟——糟到一定程度就会从悲剧翻转成喜剧。

### 机制 3：信息差幽默

**读者知道但角色不知道**：角色因为不知情而做出"认真但荒谬"的行为。
- 他以为对面是个普通老头，拼命想保护他——读者知道那个"老头"是全书最强的人
- 她偷偷给暗恋对象写情书，写得非常隐晦——读者知道对方其实已经看过了

**角色之间的信息差**：两个角色各自隐瞒着什么，读者同时知道双方的秘密，看着他们小心翼翼地试探对方。

## 三、高级技法

### Callback（回叫）

前面的小笑点在后面以更大的方式回归。第一次是一个小插曲，后来这个小插曲被反复提及、逐渐放大，最终成为一个大笑点。

规则：
- Callback 之间要有间隔（不能连续用）
- 每次回叫都要有新元素（不是简单重复）
- 最终的 Callback 要在读者不预期的时候出现

### 递进荒诞

从一个合理的起点出发，每一步都"稍微"过分一点，最后到达一个完全荒诞的位置——但因为每一步都是小幅度的，读者会在不知不觉中跟着走到荒诞。

```
他只是想买个馒头。
但馒头店关门了。隔壁包子铺也关门了。整条街都关门了。
原来今天是全城戒严——因为城主的猫丢了。
三千精兵正在挨家挨户搜查一只橘猫。
```

### "惨"比"乐"更好笑

负面情绪（倒霉、愤怒、心寒）比快乐更容易成为笑点的肥料。前提：**读者知道角色不会真的完蛋**。

- 主角越努力越倒霉——但每次都以不可思议的方式翻盘
- 角色已经够惨了，然后又发生了一件更惨的事——惨到一个临界点就会从同情翻转成爆笑

### 角色化的幽默

不同角色的幽默应该有不同的来源和表达方式：
- **智力型幽默**：通过聪明的措辞和隐喻，适合聪明角色
- **笨拙型幽默**：角色的"蠢"带来的天然喜感，但角色本人是认真的
- **自嘲型幽默**：角色拿自己开涮，显示自信或无所谓
- **毒舌型幽默**：一针见血的吐槽，适合冷面角色
- **无意识幽默**：角色完全不知道自己很好笑，旁人的反应才是笑点

## 四、节奏与剂量

### 笑点节奏

- 笑点不要均匀分布——像鞭子一样，抽一下（笑点），紧跟正经叙事（张力），再抽一下
- 大笑点之前需要"蓄力"——用 2-3 个小铺垫为大笑点做准备
- 笑点之后给读者"笑完"的时间——不要连续堆笑点，读者会麻

### 剂量控制

| 文风类型 | 笑点密度 | 笑点类型 |
|---------|---------|---------|
| 搞笑/轻松 | 每章 5-8 个笑点 | 各种类型混合 |
| 爽文 | 每章 2-4 个笑点 | 以打脸+吐槽为主 |
| 严肃+偶尔幽默 | 每 2-3 章 1 个笑点 | 冷幽默/反差，绝不能破坏氛围 |
| 纯严肃 | 几乎无笑点 | 用反讽和荒诞感替代笑点 |

### 禁区

- 紧张的生死场景中不放笑点（除非角色人设就是"用幽默应对压力"）
- 不在角色真正痛苦时制造笑点（除非是角色自己的自嘲）
- 不让笑点破坏已经建立的情绪张力
- 不在揭示重要信息时插笑话（读者的注意力会被分散）

## 五、幽默自检

每章写完，检查以下项目：

- [ ] 笑点是否有**情境支撑**？去掉情境后这个笑话还成立吗？（如果不成立，说明它依赖情境，这是好事）
- [ ] 吐槽是否有**情绪温度**？读出来是像真人在吐槽还是在做逻辑分析？
- [ ] 笑点是否与**剧情节奏**协调？有没有在不该搞笑的地方搞笑？
- [ ] 不同角色的幽默方式是否有**区分**？
- [ ] 有没有**过度解释**笑话？（讲完笑话就走，不要解释）
