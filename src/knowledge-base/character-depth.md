# 角色塑造深度指南

本文档是 Micawber（脑暴Agent）和 Wemmick（设计Agent）的核心参考，用于设计有深度、有层次的角色。

---

## 〇、人物先行原则（最高优先级）

### 核心理念：先有"人"，再有"故事功能"

角色不是为了服务主题而被"设计"出来的零件。角色是先独立存在的"人"，然后才被故事选中。

**错误流程**：主题是"复仇" → 需要一个"复仇者"角色 → 给他一段悲惨过去 → 给他复仇能力
**正确流程**：这个人叫什么？他在故事开始前过着什么生活？他有什么习惯、怪癖、执念？ → 这样的人遇到故事事件后，会自然走向什么方向？

### 人物传记法

在任何设定和属性之前，先为角色写一段 **500 字以上的人物传记**，用第三人称散文体，不用表格：

```
他出生在靠海的小镇，父亲是修船匠，母亲在鱼市帮工。六岁时
他溺过一次水，从此害怕深水但又迷恋海面的颜色。他上学迟到
时喜欢编故事而非道歉，老师很快发现他的谎话总比真话有趣。
十四岁那年镇上发生了一件事——
```

传记必须包含：
- **童年的一个具体画面**（不是"他有一个不幸的童年"，而是一个场景）
- **一个改变他认知的小事件**（不是大事件，是小到几乎没人在意的事）
- **一个他至今保留的无意义习惯**（比如洗手时数到七、把筷子对齐才肯吃饭）
- **一段他不愿提起的关系**（初恋、曾经的朋友、一个对不起的人）
- **一个他自己都不知道的行为模式**（比如每次被质疑时都会先笑）

### 心理画像法

用以下框架为角色建立内在心理结构，**但禁止把这些直接写进小说**——它们只作为行为预测的内部引擎：

| 心理维度 | 具体描述 |
|---------|---------|
| **核心恐惧** | 他最害怕什么情境（不是怕什么人/物）？ |
| **核心渴望** | 他渴望的东西是什么？为什么他一直得不到？ |
| **应对机制** | 遇到压力时的本能反应模式（逃避/攻击/讨好/冷冻） |
| **自我欺骗** | 他对自己撒的最大的谎是什么？ |
| **认知盲区** | 他看不到自己身上的什么特质（正面或负面）？ |
| **触发器** | 什么样的场景/话语会让他瞬间失控？ |

### 非功能性特征

给角色添加**与故事功能无关**的特征，让他们像真人而非棋子：

- **无用的技能**：会用口哨吹一首歌、能分辨十种咖啡豆、下棋很差但非要下
- **莫名的偏好**：讨厌圆形餐桌、只穿深色袜子、觉得某个数字不吉利
- **身体细节**：说话时喜欢摸右耳、走路时左肩微沉、笑起来牙龈会露出来
- **矛盾的小习惯**：平时邋遢但刀具摆放极其整齐、怕狗但会悄悄给流浪猫喂食

**规则**：每个主要角色至少 3 个非功能性特征。这些特征不需要服务剧情，但会自然出现在场景中，让角色"活"起来。

### 角色命名

名字不是标签，不应暗示角色的性格、职业或命运。详细规则参见 `genre-guide.md` 的"通用角色命名规则"章节。

核心要求：
- 像真人的名字，不透露角色信息
- 不要同音/同偏旁导致读者混淆
- 符合世界观和角色阶层的命名逻辑
- 不同受众类型有不同的命名风格偏好（参见 `genre-guide.md`）

## 一、角色内在矛盾设计

### 核心原则：好角色是一组矛盾

不要设计"善良的主角"或"邪恶的反派"。要设计**在特定情境下会做出让读者痛苦的选择**的角色。

### 矛盾设计框架

每个主要角色需要至少一组核心矛盾：

| 矛盾类型 | 示例 |
|---------|------|
| 信念 vs 行为 | 坚信正义，但靠谎言维持秩序 |
| 能力 vs 代价 | 力量越强，失去的人性越多 |
| 欲望 vs 需要 | 想要权力（欲望），但需要的是被接纳（需要） |
| 忠诚 vs 忠诚 | 对师父忠诚 vs 对兄弟忠诚，当两者冲突时 |
| 过去 vs 现在 | 过去的自己犯下的错，现在的自己无法逃避 |

**注意**：矛盾必须从人物传记中自然生长出来，不能凭空指定。传记写完后，回顾角色的经历，找出他身上天然形成的矛盾。

### 角色深度测试

一个角色设计完成后，用以下问题测试深度：

1. 如果把他放在另一个故事里，他的性格是否依然独特？
2. 他最不愿意承认的事情是什么？
3. 什么情况下他会违背自己的原则？
4. 他最害怕的不是什么"敌人"，而是什么"情境"？
5. 读者是否可能同时喜欢他和讨厌他？
6. **新增**：他身上有没有至少一个与故事主题完全无关的特质？
7. **新增**：他的行为模式是否可以追溯到传记中的某个具体经历？

## 二、对话个性化

### 每个角色的说话方式应该独一无二

设计对话个性时，考虑以下维度：

| 维度 | 变量 |
|------|------|
| 语速 | 话多/话少、连珠炮/慢悠悠 |
| 词汇量 | 文绉绉/大白话/行业黑话/方言 |
| 信息密度 | 直来直去/旁敲侧击/答非所问 |
| 情绪表达 | 外放/内敛/压抑/冷幽默 |
| 口头禅 | 独特的语气词或习惯用语 |
| 语法习惯 | 完整句/省略句/倒装/跑题 |

### 对话个性化示例

同一句"我不同意"，不同角色的表达：

- **军人**："不行。"（简短、命令式）
- **学者**："这个结论……恐怕证据还不够充分。"（委婉、学术腔）
- **少年**："凭什么啊！你们大人说了算？"（反抗、质疑）
- **商人**："这样吧，我们换个角度看这个问题。"（回避、转化）
- **老江湖**：（笑了笑，没说话，把茶碗推到一边）（行动代替语言）

### 对话中的潜台词

好的对话有三层含义：
1. **字面意思**：角色说了什么
2. **真实意图**：角色想表达什么
3. **情感暗流**：角色内心的真实感受

```
"饭做好了。"
"嗯，知道了。"
"凉了就不好吃了。"
"……我马上来。"
```

表面是叫人吃饭。真实意图是挽留。情感暗流是两个人之间说不出口的关心。

## 三、角色弧线设计

### 拒绝线性成长

不要：弱 → 遇到困难 → 克服困难 → 变强 → 遇到更大困难 → 变更强

要：**螺旋上升+间歇倒退**

```
阶段1：天真/自信
阶段2：第一次打击 → 怀疑
阶段3：找到新方向 → 重建信心
阶段4：更深层的打击 → 这次动摇的是根基
阶段5：看似恢复，其实改变了
阶段6：在最意想不到的时刻，过去的创伤以新形式爆发
阶段7：真正的蜕变（不是"变强"，是"变了一个人"）
```

### 角色弧的类型

不是所有角色都要"成长"：
- **正向弧**：角色克服弱点（最常见但最无聊如果处理不好）
- **负向弧**：角色被自身弱点吞噬（悲剧角色、反派堕落史）
- **平坦弧**：角色不变，但改变了周围的世界（适合信念型主角）
- **幻灭弧**：角色以为自己成长了，但读者知道他只是换了一种方式逃避

**注意**：弧线方向必须从人物传记 + 心理画像中自然推导，不能先定弧线再造角色。

## 四、反派塑造

### 核心原则：反派是"另一个世界线的主角"

反派必须有一套完整的、自洽的、甚至让读者一时间觉得"好像也有道理"的逻辑体系。

反派也必须先写人物传记，他在成为"反派"之前也是一个"人"。

### 反派深度设计清单

- [ ] 反派的动机是否源于一段真实的痛苦经历？
- [ ] 他的方法论在他的逻辑框架内是否"合理"？
- [ ] 如果故事从他的视角重写，他是否也是"主角"？
- [ ] 他是否有过犹豫或善意的瞬间？
- [ ] 他是否和主角有某种镜像关系（相似的起点，不同的选择）？
- [ ] **新增**：他是否有非功能性特征——与"邪恶"无关的日常癖好？

### 反派的层次

| 层次 | 特征 | 示例 |
|------|------|------|
| 一流 | 读者理解他甚至同情他 | 灭霸、齐天大圣（对天庭而言） |
| 二流 | 逻辑自洽但缺乏共情 | 有合理动机的冷血反派 |
| 三流 | "因为我是反派" | 纯粹为恶的黑暗领主 |

## 五、角色关系网

### 关系不是"背景"，是"冲突发动机"

每对重要角色关系都应该内含至少一个**结构性张力**：

- **师徒**：师父教的道理和现实冲突时，学生怎么选？
- **兄弟**：利益触及生存底线时，义气还在吗？
- **恋人**：当一方的成长需要远离另一方时？
- **对手**：互相成就但也互相毁灭

### 关系动态图设计

为每部小说设计一张关系网，标注：
- 每条关系线上的**核心张力**是什么
- 这条张力会在**第几章引爆**
- 引爆后关系如何**不可逆地改变**

角色关系的变化是长篇小说最重要的情感驱动力。

### 关系网涟漪检查规则

当任何角色的设定被修改时，必须执行涟漪检查：

1. 列出该角色的所有关系线
2. 逐条评估修改是否影响关系的**核心张力**
3. 如果张力改变，关联角色可能也需要调整
4. 将需要调整的关联角色及调整方向，以决策点形式呈现给用户

**禁止**静默修改关联角色 —— 所有涟漪影响必须获得用户确认。
