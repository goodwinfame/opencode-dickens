# 网络文化融入指南

本文档为 Weller（写作Agent）和 Wemmick（设计Agent）提供网络梗、俚语、谐音梗的融入策略。只有文风档案中"网络语言浓度"标注为"中"或"高"的项目才需要深度参考本文档。

---

## 一、网络梗的融入策略

### 黄金法则：调味料，不是主料

网络梗用多了发腻、用错了出戏、用旧了尴尬。好的融入是读者会心一笑，差的融入是读者觉得作者在"硬蹭"。

### 融入方式分级（从自然到生硬）

| 级别 | 方式 | 示例 | 适用浓度 |
|------|------|------|---------|
| **A级** | 角色对话中自然使用，像真人说话 | "这波啊，这波是肉包子打狗。" | 中/高 |
| **B级** | 叙述者旁白中用梗的精神而非原文 | 他的表情仿佛在说：我不是，我没有，别瞎说。 | 中 |
| **C级** | 直接引用梗的原文或格式 | 他发了条朋友圈：感谢室友不杀之恩。 | 仅高浓度 |

**原则**：优先使用 A 级和 B 级。C 级只在高浓度都市/轻喜剧中偶尔使用。

### 浓度控制

根据文风档案中的"网络语言浓度"决定使用密度：

| 浓度 | 每章使用量 | 使用范围 |
|------|----------|---------|
| **高** | 5-8 处 | 对话 + 旁白 + 章节标题均可 |
| **中** | 2-4 处 | 主要在对话中，旁白偶尔 |
| **低** | 0-1 处 | 仅限特定角色的对话 |
| **无** | 0 | 不使用任何网络语言 |

### 内容化改造

不直接搬梗，而是根据故事世界观改造梗，让梗"长在"故事里：

| 原梗 | 都市现实改造 | 修仙世界改造 |
|------|-----------|-----------|
| 社死 | 直接使用 | "道死"——在师门面前丢脸 |
| 内卷 | 直接使用 | "卷修为"——同门弟子互相攀比境界 |
| emo | 直接使用 | "入魔"——心境不稳 |
| 摆烂 | 直接使用 | "躺平悟道"——不再争抢资源 |

**关键**：改造后的梗要在故事内部逻辑上也说得通，不能只是套壳。

## 二、谐音梗设计方法

### 质量三标准

一个好的谐音梗必须同时满足：

1. **角色适配**：这个角色确实会说这种话（嘴炮角色、段子手角色适用，沉默寡言型不适用）
2. **表面成立**：不靠谐音也有字面意思，不是硬凑
3. **锦上添花**：谐音是附加笑点，不是唯一笑点。删掉谐音层，对话依然通顺

### 场景适配

| 场景类型 | 谐音梗 |
|---------|--------|
| 日常闲聊、吐槽 | 可以用 |
| 轻松打斗、斗嘴 | 可以用 |
| 紧张对峙、生死关头 | 禁用 |
| 感人告别、深情场景 | 禁用 |
| 严肃议事、庄重场合 | 禁用（但角色打破庄重的反差用法除外） |

### 谐音梗的反差用法

在庄重场合中，由特定角色突然使用谐音梗打破气氛，本身就是笑点。但此用法每个弧段最多 1 次，多了就不再有惊喜感。

## 三、网络俚语的角色适配

### 不是所有角色都说网络用语

| 角色类型 | 网络语言使用程度 | 示例 |
|---------|---------------|------|
| 年轻社交型 | 高——满嘴梗，说话自带表情包感 | "绝了""离谱""YYDS" |
| 年轻内向型 | 低——偶尔蹦出一个，显得可爱 | 平时正经，偶尔冒出"好家伙" |
| 中年角色 | 极低——用错梗反而是笑点 | 把"凡尔赛"说成"凡尔塞" |
| 老年角色 | 无——完全不用 | 用自己时代的俗语 |
| 严肃权威型 | 无——保持威严感 | 如果用了反而显得不严肃 |
| 嘴炮/段子手 | 高——这是角色标签之一 | 金句频出，梗信手拈来 |

### 反差笑点

严肃角色偶尔蹦出一句网络用语 = 天然笑点。但使用规则：
- 全书最多 2-3 次
- 必须在轻松场景中
- 用完之后角色自己或周围人要有反应（"你怎么也会说这个？"）

## 四、过时梗规避

### 核心原则

网络梗的生命周期极短（通常 3-6 个月），写进小说就变成了"时间胶囊"。优先使用"永不过时的幽默"，网络梗只作为锦上添花。

### 永不过时的幽默来源

| 类型 | 机制 | 示例 |
|------|------|------|
| **性格冲突** | 两个性格不合的人被迫合作 | 强迫症和邋遢鬼做室友 |
| **情境荒谬** | 正常人遇到不正常的情况 | 去面试发现面试官是前女友 |
| **认知反差** | 角色的认知和现实的巨大差距 | 自以为很低调但全城都知道他 |
| **错位对话** | 两个人各说各话还以为在讨论同一件事 | 一个在说工作，一个在说恋爱 |

**这些幽默类型比任何网络梗都经得起时间考验。**

### LLM 使用注意

由于 LLM 的训练数据存在时间滞后，Weller 写作时：
- 不要使用自己"不确定是否还流行"的网络梗
- 优先使用项目热梗库 `project/knowledge/trending-culture.md` 中采集的当下热梗
- 如果项目没有热梗库，只使用已成为日常用语的网络词汇（如"吐槽""打卡""种草"等已经固化的词）
- 小说设定在特定年代时，梗必须符合该年代（2010 年的角色不会说"绝绝子"）

## 五、通用幽默技巧（不依赖网络梗）

以下技巧适用于所有需要幽默元素的故事，不受时效性影响：

### 反差幽默

身份/行为反差产生的笑点——这是男性向爽文最核心的幽默来源：

```
绝世高手出门买菜被菜贩子骂了一顿不敢还嘴——
因为菜贩子是他师娘。
```

### 信息差喜剧

读者知道而角色不知道（或角色知道而其他角色不知道）的信息差：

```
所有人都在找"那个神秘高手"，只有读者知道
神秘高手正蹲在路边吃麻辣烫。
```

### 吐槽式旁白

第一人称或限制性第三人称中，叙述者/主角的内心吐槽：

```
他说他只是路过。
路过？翻了两座山、穿了一片沼泽、杀了三只妖兽的"路过"？
```

### 递进式荒诞

从正常到越来越离谱，每次递进都打破读者的预期：

```
他以为对手只有一个人。
门开了，进来十个。
又开了一扇门，又进来二十个。
他转头看窗外——外面还排着队。
```

### 冷幽默/黑色幽默

用平淡的语气说不平淡的事：

```
"你杀了多少人？"
"没数过。我数学不好。"
```

## 六、项目热梗库（运行时生成）

对于网络语言浓度为"中"或"高"的项目，Dickens 会在 Phase 1 结束后从网络采集当下流行的梗/俚语/流行表达，存入项目目录 `project/knowledge/trending-culture.md`。

### 热梗库格式

```markdown
# 当下流行网络文化（采集于 YYYY-MM-DD）

## 热门梗/流行语
| 梗/表达 | 含义 | 来源 | 适用场景 | 注意事项 |
|---------|------|------|---------|---------|
| [梗1]   | [释义] | [出处] | [轻松对话/吐槽/...] | [使用禁忌] |

## 当下流行俚语
| 俚语 | 含义 | 使用人群 |
|------|------|---------|

## 谐音梗素材
| 原词 | 谐音 | 笑点 | 使用限制 |
|------|------|------|---------|

## 已过时/敏感（避免使用）
| 梗 | 过时原因 |
|-----|---------|
```

### 使用规则

- 热梗库是**素材池**，不是"必须使用清单"——Weller 按需取用，不强制堆砌
- 每条梗都标注适用场景和使用限制
- "已过时"列表防止 Weller 凭模型记忆使用已退潮的梗
- 每个新弧段开始前可选择性刷新（长篇写作跨度可能数月，梗会更新换代）
