# 情节设计与爆点引爆指南

本文档是 Micawber（脑暴Agent）、Wemmick（设计Agent）和 Dickens（编排Agent）的参考，用于设计有爆发力的情节结构。

---

## 一、爆点类型学

### 什么是"爆点"

爆点 = 读者读到这里时的**强烈情绪反应**（震惊、狂喜、心碎、热血沸腾）。不是"发生了大事"，而是"读者的情绪被引爆"。

### 爆点分类

| 类型 | 机制 | 示例模式 |
|------|------|---------|
| 反转爆点 | 颠覆读者认知 | "原来一直以来的盟友才是真正的幕后黑手" |
| 揭示爆点 | 揭开关键秘密 | "主角的身世真相"、"世界的真实面貌" |
| 情感爆点 | 压抑的情感总爆发 | 沉默的角色终于开口说出真心话 |
| 能力爆点 | 关键时刻能力觉醒/突破 | 绝境中解锁新能力，但要有足够的铺垫 |
| 选择爆点 | 两难困境下的艰难抉择 | 必须在两个珍贵的人之间选一个 |
| 牺牲爆点 | 有意义的牺牲 | 角色为了更高目标放弃最珍贵的东西 |
| 因果爆点 | 早期行为的延迟后果 | 100章前的一个小决定，现在引发了灾难 |

### 爆点设计原则

1. **爆点必须有铺垫**：没有铺垫的反转是廉价的。铺垫越长，爆发越强。
2. **爆点后必须有余波**：爆点不是终点，是新阶段的起点。
3. **不是所有爆点都要"大"**：安静的情感爆点有时比打斗型爆点更有冲击力。
4. **爆点间隔要有节奏**：连续爆点会让读者麻木。

## 二、伏笔设计

### 三层伏笔法

每个重要揭示至少设置三层伏笔：

**第一层：表面线索**（读者第一次读可能注意到但不在意）
- 看似随意的细节描写
- 角色的一句"闲话"
- 背景中一个不起眼的物品

**第二层：误导层**（读者注意到了，但被引向错误的方向）
- 提供一个"合理"但错误的解释
- 让另一个角色对线索做出错误解读
- 用明显的线索掩盖真正的线索

**第三层：真相揭示**（回看前文，所有细节都指向这个答案）
- 揭示后，读者回想前文会感叹"原来如此"
- 所有"不合理"的细节突然全部合理

### 伏笔铺设时间

| 爆点级别 | 最短铺垫距离 | 建议铺垫距离 |
|---------|------------|------------|
| 小爆点 | 1-2章 | 3-5章 |
| 中爆点 | 5-10章 | 10-20章 |
| 大爆点 | 20章+ | 50-100章 |
| 终极爆点 | 全书铺垫 | 从第一章开始 |

### 伏笔管理

使用 `dickens_consistency` 工具追踪每条伏笔：
- 铺设章节和内容
- 计划揭示章节
- 当前状态（铺设中/部分揭示/已揭示）
- 关联的其他伏笔

## 三、张力积累："压力锅"模型

### 原理

故事的张力像压力锅：
1. **加压**：制造冲突、增加障碍、提升赌注
2. **晃动**：给出希望又夺走、制造小的释放（假爆点）
3. **爆发**：压力到达临界点，真正的爆点引爆
4. **泄压**：爆发后的短暂平静，让读者喘息

### 加压手段

- **时间压力**：倒计时、deadline
- **信息不对等**：读者知道危险但角色不知道
- **多线交汇**：多条支线同时到达临界点
- **道德压力**：每个选择都有代价
- **损失叠加**：角色不断失去东西

### 节奏公式

```
紧-紧-紧-松 = 基本单元
小爆点 → 喘息 → 加压 → 中爆点 → 喘息 → 高强度加压 → 大爆点

弧段级别的节奏曲线：
前3章：建立 → 中间章节：层层加压 → 倒数2章：到达极限 → 最后1章：爆发+余波
```

## 四、信息差的运用

### 信息差 = 张力制造器

| 类型 | 效果 | 运用场景 |
|------|------|---------|
| 读者 > 角色 | 悬念、紧张（"别进那个房间！"） | 恐怖、悬疑、悲剧预兆 |
| 角色 > 读者 | 好奇、期待（"他到底在计划什么？"） | 智斗、反转铺垫 |
| 角色A > 角色B | 权力动态、操纵感 | 宫斗、策略、背叛线 |
| 读者 = 角色 | 共同发现的震撼 | 探险、揭秘 |

### 信息差管理

在大纲阶段就标注清楚：
- 每章结束时，每个角色各知道什么？
- 读者知道什么？
- 哪些信息差会在何时被打破？

## 五、结构选择

### 不只是"三幕结构"

根据故事类型选择最适合的结构：

| 结构 | 适用类型 | 特点 |
|------|---------|------|
| 三幕式 | 通用 | 建立-对抗-解决，经典但容易老套 |
| 螺旋上升 | 修仙/成长 | 每个弧段重复"挑战-成长"但层级递进 |
| 多线交织 | 群像/史诗 | 多条故事线交替推进，在关键点交汇 |
| 倒叙解谜 | 悬疑/推理 | 先给结果，再拆解过程 |
| 拼图式 | 复杂叙事 | 碎片化叙事，读者在阅读过程中拼出全貌 |
| 循环式 | 哲理/宿命 | 故事的结尾呼应开头，但含义已经完全不同 |

### 弧段设计

长篇小说的弧段（Arc）是核心节奏单元：

- 每个弧段有自己的**小故事弧**（有开端、发展、高潮、结局）
- 弧段之间有**递进关系**（赌注越来越高）
- 弧段的高潮是**全书级别爆点的子集**
- 弧段末尾需要**钩子**（让读者继续读下一个弧段）

## 六、章末钩子设计

### 钩子类型

| 类型 | 效果 | 示例 |
|------|------|------|
| 悬念型 | "然后呢？" | 新威胁出现、关键选择悬而未决 |
| 揭示型 | "什么？！" | 章末一句话颠覆前文认知 |
| 情感型 | "不要……" | 角色做出令人心痛的决定 |
| 场景切换型 | "另一边呢？" | 在高潮处切到另一条线 |
| 静默型 | "……" | 以沉默、空白、意味深长的画面结束 |

### 钩子密度

不是每章都需要强钩子。建议：
- 每3-5章有1个强钩子
- 弧段结尾必须有强钩子
- 中间章节可以用温和的悬念或情感余韵结尾
- 偶尔用"静默型"结尾制造节奏变化
